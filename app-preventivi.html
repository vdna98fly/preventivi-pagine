Collegamento GitHub — dati condivisi e pubblicazione

Salvato solo in questo browser (localStorage). Non finisce mai nei file sui repository.
Guida al setup GitHub (una volta sola, 10 minuti)
  1. Crea i due repository su github.com/new: uno privato per i dati (es. preventivi-dati) e uno pubblico per le pagine cliente (es. preventivi-pagine). In entrambi spunta "Add a README" così il branch main esiste subito.
  2. Attiva GitHub Pages sul repo pubblico: Settings → Pages → Source "Deploy from a branch" → branch main, cartella / (root) → Save.
  3. Crea il token: github.com → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token. Repository access: "Only select repositories" → seleziona solo i due repo. Permissions → Repository permissions → Contents: Read and write. Scadenza a piacere (alla scadenza basta rigenerarlo e reinserirlo qui).
  4. Incolla qui token, utente e nomi dei repository, poi premi "Collega e verifica": l'app testa l'accesso e fa il primo allineamento dei dati.
  5. Sugli altri PC/tablet dei venditori: apri l'app, incolla lo stesso token (o uno dedicato) e premi di nuovo "Collega e verifica". Da quel momento archivio e catalogo sono condivisi.
Le pagine pubblicate saranno su https://<utente>.github.io/<repo-pagine>/p/<codice>.html. Il codice contiene un suffisso casuale: solo chi riceve il link può aprirle.
